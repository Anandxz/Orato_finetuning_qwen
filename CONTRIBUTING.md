# Contributing

- Keep each branch or change focused on one milestone.
- Run the relevant tests before committing and report the commands actually run.
- Review and justify dependency changes, especially heavy training packages.
- Never commit credentials, private data, audio, datasets, weights, or checkpoints.
- Keep authoritative source code in Python modules and command-line entry points,
  not notebooks.
- Update documentation whenever user-visible behaviour changes.
- Avoid unrelated features and refactors.
- Do not claim successful execution without evidence from the relevant command.
